3 sleepyMUSTACHE
4 ================================================================================
5 
6 Doxygen [Documentation] (http://www.sleepymustache.com/documentation/html/index.html) is available.
7 
8 sleepyMUSTACHE is a PHP framework that comes with solutions for everyday php
9 challenges. Most of the functionality is optional and tries to be as minimalist
10 as possible.
11

13 --------------------------------------------------------------------------------

15 ### Core Functionality
16 
17 The core is the basic functions that modules build off of
18 
19 * **Debugging** -
20  Easily send debug information via the browser, email, or database
21 
22 * **Hooks** -
23  Assign functions to run at specific hook points throughout the loading

115 Sample Code
116 --------------------------------------------------------------------------------
117 ### Hooks
118 
119 The *Hooks* system is made up of *hook filters* and *hook actions*. *Hook
120 actions* are points in the code where you can assign functions to run. For
121 example, we can put a *hook action* after a record is saved to the database,
122 then assign a function to the *hook action* that will send an email after
123 the DB update.
124 
125  // Save to the database
126  $db->save();
127 
128  // add a hook action
129  Hook::addAction('record_saved');
130 
131  // Add a function to the hook action
132  function send_email() {
133  // send an email saying a record was updated
134  }
135 
136  Hook::doAction(
137  'record_saved',
138  'send_email'
139  );
140 
141 *Hook filters* are similar to *hook actions* but pass data as
142 parameters to the functions that get assigned to the hook. After manipulating
143 this data you should return the edited data back to the program.
144 
145  // add a hook filter
146  $content = Hook::addFilter('update_content', $_POST['content']);
147 
148  // Add a function to the hook filter
149  function clean_html ($html) {
150  $c = htmlentities(trim($html), ENT_NOQUOTES, "UTF-8", false);
151  return $c;
152  }
153 
154  Hook::applyFilter(
155  'update_content',
156  'clean_html'
157  );
158 
159 The *modules/enabled* directory provides a convenient location to put code that
160 utilized the hooks system. Code inside of the *modules/enabled* directory are
161 automatically added to the program at runtime.

164 ### Templating
165 
166 Templates reside inside the *'/templates/'* folder and should end in a .tpl
167 extension. The templating system works by using placeholders that later get
168 replaced with text. The placeholders must have the following syntax:
169 
170  {{ placeholder }}
171 
172 To use a template you instantiate the template class passing in the template
173 name. You then bind data to the placeholders and call the *Template::show()*
174 method.
175 
176  require_once('include/sleepy.php');
177 
178  $page = new Template('default');
179  $page->bind('title', 'sleepyMUSTACHE');
180  $page->bind('header', 'Hello world!');
181  $page->show();
182 
183 Here is the sample template file (templates/default.tpl)
184 
185  <html>
186  <head>
187  <title>{{ title }}</title>
188  </head>
189  <body>
190  <h1>{{ header }}</h1>
191  <p>This page has been viewed {{ hits }} times.</p>
192  </body>
193  </html>
194 
195 We added a *{{ hits }}* placeholder in the template above. For this example, we
196 want to replace the placeholder with the number of times this page was viewed.
197 We can add that functionality using Hooks.
198 
199  // filename: /modules/enabled/hit-counter/hits.php
200  function hook_render_placeholder_hits() {
201  $hits = new FakeClass();
202 
203  return $hits->getTotal();
204  }
205 
206  Hook::applyFilter(
207  'render_placeholder_hits',
208  'hook_render_placeholder_hits'
209  );
210 
211 The first parameter of *Hook:applyFilter*, the hook filter, ends in 'hits' which
212 correlates to the name of the placeholder. This hook filter is defined in
213 '*class.template.php*'. The second parameter is the name of the function to run
214 when we render the placeholder.
215 
216 You can iterate through multidimensional array data using #each placeholders
217 
218  // Bind the data like this
219  $page->bind('fruits', array(
220  array(
221  "name" => "apple",
222  "color" => "red"
223  ), array(
224  "name" => "banana",
225  "color" => "yellow"
226  )
227  ));
228 
229  // in the template
230  {{ #each f in fruits }}
231  <p>I like {{ f.color }}, because my {{ f.name }} is {{ f.color }}.</p>
232  {{ /each }}

234 ### Databases
235 
236 The database connection settings are defined in the */include/global.php* file.
237 After the *LIVE_URL* is set in *global.php* the framework will detect which DB
238 to use based on the current URL.
239 
240 To get a database instance, use:
241 
242  $db = DB::getInstance();
243 
244 The DB class is static and will automatically handle suppressing multiple
245 instances.

247 ### Sending emails
248 
249 The Mailer class simplifies sending emails by generating headers for you
250 and using an easy to use object to clearly define your email. The Mailer can
251 send emails using an HTML template or text.
252 
253  $m = new Mailer();
254  $m->addTo("test@test.com");
255  $m->addFrom("from.me@test.com");
256  $m->addSubject("This is a test, don't panic.");
257  $m->fetchHTML("http://test.com/template.php");
258  // OR
259  $m->msgText("This is my message.")
260  $m->send();

291 ### File System Database (class.fsdb.php)
292 
293 Sometimes using a database is overkill. A simple solution is to use the *FSDB*.
294 It is very simple and does not allow complex queries, however it is fast, easy
295 to use, and requires no setup, except checking that proper permissions are set.
296 
297  $fruit = new stdClass();
298 
299  $fruit->name = "Apple";
300  $fruit->color = "Red";
301  $fruit->texture = "Crispy";
302  $fruit->price = 0.50;
303 
304  $db = new FSDB();
305 
306  $db->insert('fruit', $fruit);
307  $data = $db->select('fruit', 'name', 'Apple');

334 ### Navigation
335 
336 The navigation is generated from JSON. It renders the JSON into a unordered
337 list with some classes added for the current active page.
338 
339  // Add a placeholder in your template
340  {{ TopNav }}
341 
342  // Create a php file in */modules/enabled/*
343  require_once('include/class.navigation.php');
344 
345  // create a function to add to the *hook filter*
346  function hook_render_placeholder_TopNav() {
347 
348  // Page data is passed via JSON
349  $topNavData = '{
350  "pages": [
351  {
352  "title": "Nav 1",
353  "link": "/nav1/"
354  }, {
355  "title": "Nav 2",
356  "link": "/nav2/",
357  "pages": [
358  {
359  "title": "Subnav 1",
360  "link": "/downloads/fpo.pdf",
361  "target": "_blank"
362  }
363  ]
364  }
365  ]
366  }';
367 
368  $topNav = new Navigation($topNavData);
369  $topNav->setCurrent($_SERVER['SCRIPT_NAME']);
370 
371  return $topNav->show();
372  }
373 
374  Hook::applyFilter(
375  'render_placeholder_TopNav',
376  'hook_render_placeholder_TopNav'
377  );
